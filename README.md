# Telegram Bot - Маркетплейс

Telegram бот для marketplace с каталогом товаров, управлением заказами и платежной интеграцией.

## 🚀 Возможности

### Бизнес-функции
- **Маркетплейс**: Каталог товаров с категориями
- **Управление заказами**: Полный жизненный цикл заказов со статусами
- **Платежи**: Поддержка Crypto Bot
- **Админ-панель**: Полный административный контроль
- **Управление пользователями**: Регистрация, профили, система баланса
- **Промокоды**: Система скидок с лимитами использования
- **Криптоплатежи**: Автоматическая обработка инвойсов

### Архитектура
- **Многослойный дизайн**: Разделение на уровни (Application → Domain → Data)
- **Готов к VPS**: Оптимизирован для развертывания на одном сервере
- **Systemd интеграция**: Автозапуск и перезапуск при ошибках
- **Логирование**: Комплексное ведение логов

## 📋 Требования

- Python 3.9+
- SQLite (или PostgreSQL)
- 1GB RAM минимум (2GB рекомендуется)
- 20GB дисковое пространство

## 🔧 Быстрый старт

### Установка

```bash
# Клонирование репозитория
git clone <repository-url>
cd mp

# Создание виртуального окружения
python3 -m venv venv
source venv/bin/activate  # На Windows: venv\Scripts\Activate

# Установка зависимостей
pip install -r requirements.txt

# Настройка окружения
cp .env.example .env
nano .env  # Добавьте ваш BOT_TOKEN и другие настройки
```

### Запуск

```bash
# Запуск бота
python main.py
```

## 🏗️ Архитектура

### Структура уровней

```
Application Layer (handlers/)     → Обработка событий Telegram
Domain Layer (services/, core/)   → Бизнес-логика
Data Layer (db/)                  → Абстракция базы данных
```

## 🛠️ Конфигурация

### Переменные окружения

Создайте файл `.env` на основе `.env.example`:

```bash
# Обязательно
BOT_TOKEN=your_bot_token
ADMIN_IDS=123456789,987654321

# Опционально
DATABASE_URL=sqlite+aiosqlite:///data/database.db

CRYPTO_ENABLED=true
CRYPTO_TOKEN=your_crypto_bot_token
REVIEWS_CHANNEL=https://t.me/your_reviews_channel
LOG_LEVEL=INFO
```

```bash

sudo journalctl -u telegram-bot -f

sudo journalctl -u telegram-bot -n 50
```

## 📊 Система заказов

### Статусы заказов

- **PENDING**: Заказ создан, ожидает оплаты
- **CONFIRMED**: Оплата подтверждена, ожидает администратора
- **COMPLETED**: Заказ доставлен пользователю
- **CANCELLED**: Заказ отменен

### Функции пользователей

- **"Мои заказы"**: Просмотр истории заказов с группировкой по статусам
- **Детали заказа**: Отслеживание прогресса заказа
- **Методы оплаты**: Crypto Bot

## 🔒 Рекомендации по безопасности

### Развертывание

- ✅ Используйте переменные окружения для всех секретов
- ✅ Установите права на файл: `chmod 600 .env`
- ✅ Запускайте от имени отдельного пользователя (не root)
- ✅ Включите firewall (ufw/iptables)
- ✅ Регулярные обновления безопасности

### Разработка

- ✅ Никогда не коммитьте `.env` файл
- ✅ Тестируйте перед развертыванием
- ✅ Обновляйте зависимости
- ✅ Регулярно просматривайте логи

## 🔄 Обслуживание

### Обновления

```bash
# Получение последних изменений
git pull origin main

# Перезапуск сервиса
sudo systemctl restart telegram-bot
```

### Резервное копирование базы данных

```bash
# Ручное резервное копирование
sqlite3 data/database.db ".backup data/database_backup_$(date +%Y%m%d).db"

# Автоматическое резервное копирование (добавить в crontab)
0 2 * * * sqlite3 /opt/telegram-bot/data/database.db ".backup /opt/backups/db_$(date +\%Y\%m\%d).db"
```

## 🐛 Устранение неполадок

### Сервис не запускается

```bash
# Проверка статуса сервиса
sudo systemctl status telegram-bot

# Просмотр подробных логов
sudo journalctl -u telegram-bot -n 100 --no-pager
```
